
## 最近对问题
### 理解问题
* 找出一个包含n个点的集合中距离最近的两个点。
### 选择策略
* 分别计算每一点对之间的距离，然后从中找出距离最小的那一对。为了避免同一点对计算两次，可以只考虑i<j的点对(Pi, Pj)
### 算法设计
* 算法 bruteForceClosesPoints(P)
```
//蛮力法求解平面中距离最近的两点
//输入：一个n(n≥2)个点的列表P，P1=(x1, y1)，…，Pn=(xn, yn)
//输出：两个最近点的下标
dmin←∞
for i←0 to n-2 do
  for j←i+1 to n-1 do
    d←(xi-xj)2+(yi-yj)2
    if d<dmin
        dmin←d; index1←i; index2←j;
return index1,index2
```
### 正确性证明
### 算法分析
$O(n^2)$
